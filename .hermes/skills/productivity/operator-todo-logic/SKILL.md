# Operator TODO Logic (Hermes)

## Overview
Hermes persists a durable, cross-session operator TODO/hold queue for critical, deferred, or multi-session tasks. All entries are templated (overview/problem/actions so far, next/notes). The index lives at ${HOME}/OPERATOR_TODO.md and is always anchored in the Knowledge Base operator-index.

## Usage
- Add an operator TODO: instruct Hermes to add or update a task (details auto-templated, synchronized to both standard locations).
- Surface or clear: request the TODO summary or details any time; update via specific instructions.

## Paths
- Local file: ${HOME}/OPERATOR_TODO.md
- Repo mirror: ${WORKSPACE_ROOT}/hermes-memory-wiki/operator-index.md (regenerated by scripts/generate_vault.py on wiki rebuild)
- Web-visible Knowledge Base note: ${WORKSPACE_ROOT}/hermes-memory-wiki/vault/documents/operator-todo-index.md

## Verification
- The local queue, repo mirror, and web-visible Knowledge Base note exist and have synchronized TODO entries (first templated entry is checked after each edit).
- `scripts/generate_vault.py` regenerates both `${WORKSPACE_ROOT}/hermes-memory-wiki/operator-index.md` and `${WORKSPACE_ROOT}/hermes-memory-wiki/vault/documents/operator-todo-index.md` from `${HOME}/OPERATOR_TODO.md` on rebuild.
- After generator or TODO-flow changes, rebuild Knowledge Base and run a focused ad-hoc verification script under `/tmp/hermes-verify-*.sh` to check: canonical queue, repo mirror, generated vault note, built HTML, live route, and cron prompt references.
- Morning Executive Brief may surface active operator TODOs by reading ${HOME}/OPERATOR_TODO.md.

## Pitfalls
- Manual edits to `vault/documents/operator-todo-index.md` are not durable by themselves; rebuilds can wipe them unless `scripts/generate_vault.py` is updated to regenerate the note.
- If editing both local and mirror layers at once, specify which file/section is canonical to avoid overwrite confusion.
- If either generated mirror is missing after rebuild, fix the generator first rather than repeatedly rewriting the output file.
